# Tour of Duty — Missão com Prazo

Framework para definir ciclos de carreira com missão, duração e critério de sucesso claros.

> Origem: Reid Hoffman, Ben Casnocha, Chris Yeh — "The Alliance" (2014).

## O Conceito

Uma Tour of Duty é um **compromisso mútuo de prazo definido** entre o profissional e a organização. Em vez de pensar em "emprego indefinido", ambos definem uma missão com início, fim e resultado esperado.

Cada tour tem:
- **Missão:** o que o profissional vai realizar
- **Duração:** 2-4 anos (típico)
- **O que o profissional ganha:** experiência, rede, habilidades, credencial
- **O que a organização ganha:** resultado específico, impacto mensurável
- **Critério de sucesso:** como saber que a tour foi bem-sucedida

## Tipos de Tour

| Tipo | Duração | Perfil | Exemplo PM |
|---|---|---|---|
| **Rotational** | 1-2 anos | Exploratório, fase inicial de carreira | PM Jr rodando em diferentes squads para encontrar fit |
| **Transformational** | 2-4 anos | Missão específica de alto impacto | PM liderando a transição de um processo manual → automatizado |
| **Foundational** | Indefinido | Alinhamento profundo com a missão da empresa | PM Sênior que é co-criador da estratégia de produto |

## Quando Usar

- PM quer **transição de papel** (ex: de PM para liderança, de PM para Staff)
- PM quer **experiência específica** (ex: liderar produto de dados, construir time)
- PM está no mesmo papel há 2+ anos e precisa definir o próximo ciclo
- Conversa de carreira com o gestor: definir o que os próximos 2 anos significam

## Como Aplicar

### 1. Diagnosticar a tour atual

Perguntar:
- "O que você está fazendo hoje se encaixa em qual tipo de tour?"
- "Quando começou essa tour? Quando deveria terminar?"
- "O que você combinou (implícita ou explicitamente) com seu gestor quando entrou nesse papel?"

Se não há tour definida → a pessoa está em "emprego indefinido", que é o anti-pattern principal.

### 2. Definir a próxima tour

Preencher:

| Campo | Resposta |
|---|---|
| **Missão** | [O que vou realizar — específico, não vago] |
| **Tipo** | Rotational / Transformational / Foundational |
| **Duração** | [X meses ou anos] |
| **O que ganho** | [habilidades, experiência, rede, credencial] |
| **O que a organização ganha** | [resultado mensurável] |
| **Critério de sucesso** | [como saber que deu certo — métrica ou evidência] |
| **O que acontece depois** | [próxima tour, promoção, nova área, saída] |

### 3. Alinhar com o gestor

A tour só funciona como **acordo bilateral**. O PM define a proposta, discute com o gestor, e ambos se comprometem.

**Perguntas para o gestor:**
- "Essa missão faz sentido para os objetivos da área?"
- "Você vai me dar espaço e suporte para essa missão?"
- "O que acontece se eu entregar? E se não entregar?"

### 4. Revisar no meio da tour

A cada 6 meses:
- A missão ainda faz sentido?
- O prazo precisa de ajuste?
- O critério de sucesso mudou?

## Anti-Patterns

| Anti-Pattern | Problema | Correção |
|---|---|---|
| Tour vaga | "Crescer como PM" não é missão | Definir entregável e métrica específicos |
| Tour infinita | Sem prazo = sem urgência | Definir data de acompanhamento e de conclusão |
| Tour unilateral | Só o PM sabe, gestor não comprou | Formalizar como acordo bilateral |
| Medo de definir o "depois" | Evitar a conversa sobre o que vem depois | O "depois" é o motivador — sem ele, a tour perde energia |

## Exemplo Completo

**PM Pleno querendo chegar a Sênior:**

| Campo | Resposta |
|---|---|
| **Missão** | Liderar uma iniciativa estratégica do zero ao lançamento, com impacto mensurável em uma métrica-chave do produto |
| **Tipo** | Transformational |
| **Duração** | 18 meses (Q2/2026 — Q3/2027) |
| **O que ganho** | Experiência em liderar iniciativa estratégica cross-time, comunicação com diretoria, experimentação com dados |
| **O que a organização ganha** | Métrica-chave batendo a meta (ex.: > X% de automação, redução do SLA do processo) |
| **Critério de sucesso** | Feature em produção + métricas batendo target por 2 meses consecutivos + apresentação de resultado para diretoria |
| **O que acontece depois** | Promoção a Sênior se critérios do career-ladder forem atendidos, ou nova tour com escopo ampliado |

## Combinação com Outros Frameworks

- **Com 70-20-10:** a tour define O QUÊ, o 70-20-10 define COMO desenvolver as competências necessárias durante a tour
- **Com Competency Roadmap:** o gap do PMwheel aponta QUAL competência, a tour dá o contexto prático para desenvolvê-la
- **Com GROW:** usar GROW para definir a missão da tour quando o PM está confuso sobre o que quer
