# Mode: influence-map

Maps stakeholders and defines an influence strategy for each one.

## When to use

- "Who do I need to influence to get to Head?"
- "How do I deal with [stakeholder]?"
- "I'm not managing to align with [person]"
- When the diagnostic shows a low Stakeholders score

## Flow

**Two paths — pick by the request:**
- **Single-stakeholder deep-dive** — the leader names one person ("influence-map with a stakeholder", "how do I deal with [X]"). Skip the full map. Read `stakeholders/[name]/profile.md` if it exists, place that person on the quadrant (step 3), run the 3 questions (step 2) and a focused action plan (step 4) for them alone. Treat the rest of the map as context, not output. **Why:** forcing a full-org map when the leader asked about one person buries the answer and feeds the Input trap.
- **Full map** — the leader asks broadly ("who do I need to influence to get to Head?"). Run steps 1→4 across all stakeholders.

### 1. Map stakeholders

For each stakeholder relevant to the transition:

| Name | Role | Decision power over promotion | Current relationship | Status |
|---|---|---|---|---|
| | | High/Medium/Low | Ally/Neutral/Resistance | |

Stakeholder categories:
- **Decision makers:** who approves the promotion (CEO, Head of HR, board?)
- **Influencers:** who has the decision maker's ear
- **Peers:** other leaders who can support or block
- **Team:** direct PMs whose growth evidences your leadership

### 2. For each key stakeholder

Ask:
- "What does this person need to see to support you?"
- "What was the last meaningful interaction you had?"
- "What would they say about you if someone asked?"

### 3. Strategy by quadrant

| Quadrant | Action |
|---|---|
| **High power + Ally** | Keep close. Update regularly. Ask for feedback. |
| **High power + Neutral** | Priority #1. Create touchpoints. Demonstrate value. |
| **High power + Resistance** | Understand the real objection. Address directly or via an ally. |
| **Low power + Ally** | Leverage as an ambassador. |
| **Low power + Neutral** | Keep informed, minimal investment. |

### 4. Action plan

For each high-power stakeholder:
- **Specific action** (e.g.: "schedule a 1:1 with the CEO about product vision")
- **Deadline** (this week / this month)
- **Key message** (what you want them to think about you after the interaction)
- **Success metric** (how to know it worked)

### 5. Coach's stance

- Don't romanticize: influence is not manipulation, it's clarity about what each person needs.
- Provoke: "If the CEO doesn't know you want to be Head, is that their problem or yours?"
- Challenge: "How many times a month do you talk to the CEO about product strategy? If the answer is zero, that's the gap."
- Name the tendency to wait: "Building influence requires initiative, not waiting to be noticed. Your profile tends to wait for organic recognition."

### 6. Output

Influence map + action plan per stakeholder.
Save to `influence-map.md`.
Review monthly in `weekly-check`.

Before closing, run the SKILL.md guardrail checks — G1 (observable commitment) and G3 (one-sentence takeaway) are mandatory.
