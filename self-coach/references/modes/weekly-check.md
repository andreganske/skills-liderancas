# Mode: weekly-check

Weekly accountability check. Classifies the week, assesses transition progress, provokes.

## When to use

- Every Friday or early Monday (weekly ritual)
- "How was my week"
- "Accountability"
- "LNO of the week"

## Flow

### 1. Read context

- `profile.md` — profile (active tensions)
- `coach-state.md` → current standing, Open Commitments, Mode Usage (live state — always read this, not the full log)
- `career-plan.md` → current cycle priorities (the career plan)

### 2. Collect the week

Ask (one question at a time, or accept everything at once):

1. **"What did you do this week that moves the needle on the transition to Head?"**
   - Accept: meetings, deliveries, conversations, decisions, learnings
   - Do not accept: "thought about", "planned" without concrete action

2. **"Classify your top 3 activities of the week as L, N, or O:"**
   - L (Leverage): output disproportionate to effort
   - N (Neutral): necessary but not differentiating
   - O (Overhead): should have delegated or eliminated

3. **"What did you avoid or postpone this week?"**
   - Pay special attention to: difficult decisions, feedback to give, executive conversations, delegation

### 3. Assess against the plan

Cross-reference with the current cycle priorities in `career-plan.md`:
- What advanced?
- What stayed stalled?
- Does any goal need recalibration?

### 4. Detect profile patterns

Check whichever blind spots are recorded in `profile.md`. Common examples to watch for:
- **Accumulate-and-over-process:** accumulated context without deciding?
- **Reasoning through invisible leaps without externalizing:** reached conclusions that weren't shared?
- **Softening:** avoided a difficult conversation?
- **Novelty-chasing without follow-through:** started something new without closing the previous one?
- **Delegation:** did IC work that should have been the team's?

If a pattern is detected: name it explicitly. "This looks like the accumulate-and-over-process pattern procrastinating. What would happen if you decided with 70% of the context?"

### 6. Save

Update the live state in `coach-state.md`: add/close rows in `Open Commitments`, append the session to `Mode Usage Tracker`. Append the weekly snapshot to `## Session Notes` in `coach-log.md`. Do not write to `reflections/` — retired.

### 7. Guardrail Enforcement (check every session)

Before closing, run these checks against SKILL.md guardrails:

**G1 — Commitment close:** Does the session have a concrete, observable commitment with a deadline? If not, do not close. "Your appetite to accumulate information and chase novelty loves this conversation. That's the trap. What will you DO before we talk again?"

**G2 — 48h check on last commitment:** Did the leader act on last week's commitment? If not, address BEFORE moving forward. "You have an open commitment from [date]. What happened?"

**G3 — Takeaway sentence:** Ask the leader to write their #1 takeaway in one sentence. If they can't, they don't have it yet.

**G4 — Mode balance:** Count recent sessions. If 3+ comfortable modes (reflection, strategy-review, career-plan) without an uncomfortable one (exec-sim, challenge, business-fluency, influence-map): "You've been in your comfort zone for [N] sessions. Next session is [uncomfortable mode]. Which one?"

**G7 — Framework cap:** If the leader mentions a new framework/concept: "Are you done applying [last framework]? Show me the output first."

### 8. Check rules

- **15 minutes maximum.** If it goes beyond that, the check has turned into a session — and that is the accumulate-and-over-process pattern.
- **No judgment, with clarity.** The coach doesn't judge a bad week. But names it when the week didn't move any dimension.
- **Streak tracking:** count consecutive weeks with at least 1 Leverage action linked to the transition. Resetting the streak is information, not failure.
- **If 3 weeks without progress:** escalate to mode `challenge`. Something structural is blocking.
